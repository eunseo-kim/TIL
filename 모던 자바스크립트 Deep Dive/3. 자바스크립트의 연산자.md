# 📝자바스크립트의 연산자(7장) / TIL(21.02.28)
## 📌 연산자
- `+` 연산자 : 서로 다른 타입을 `+` 으로 연결할 경우 자바스크립트 엔진에 의해 암묵적으로 타입이 자동 변환되기도 한다. 이를 **암묵적 타입 변환** 또는 **타입 강제 변환**이라고 한다.
```js
// null은 0으로 타입 변환된다.
1 + null;		// 1
// undefined는 숫자로 타입 변환 되지 않는다.
1 + undefined;	// NaN
```
- 할당문은 값으로 평가되는 표현식인 문으로, 할당된 값으로 평가될 수 있다.
```js
var x;
// 할당문은 표현식인 문이다.
console.log(x = 10);	//10 
```
- `typeof` 연산자 : `typeof`로 `null` 값을 연산해보면 `null`이 아닌 `object`를 반환한다. 이것은 자바스크립트의 첫번째 버그이다. 따라서 **값이 `null`인지 확인하려면 `typeof`를 사용하지 않고 일치 비교 연산자`===`를 사용**하자.
```js
var foo = null;
typeof foo === null;	// false
foo === null;			// true
```

### 💡 동등/일치 비교 연산자
- 동등 비교 연산자 `==` : 좌항과 우항의 피연산자를 비교할 때 먼저 **암묵적인 타입 변환**을 통해 타입을 일치시킨 후 같은 값인지 비교한다. 따라서 결과를 예측하기 어렵고 실수하기 쉽다. 
```js
5 == 5;			// true
5 == '5';		// true
```
- 일치 비교 연산자 `===` : 좌항, 우항의 피연산자가 타입도 같고 값도 같은 경우에 한하여 true를 반환한다. 즉, **암묵적 타입 변환 없이** 값을 비교한다. 결과를 예측하기 쉽다. 
- 따라서 동등 비교 연산자의 사용은 지양하고, 되도록 일치 비교 연산자를 사용하는 것을 권장한다.
```js
5 === 5;		// true
5 === '5';		// false
```
### 🔥일치 비교 연산자에서 주의할 점🔥
- `NaN`은 자신과 일치하지 않는 유일한 값이다. 따라서 **숫자가 `NaN`인 지 조사하려면 빌트인 함수인 `isNaN`을 사용**해야 한다. 
- 숫자 0도 주의하자. **일치 비교 연산자로는 `+0`과 `-0`을 비교해도 true를 반환**한다.
- **ES6에 도입된 `Object.is`를 사용하면 위의 `NaN`문제와 `+0, -0` 문제를 해결**할 수 있다. `Object.is`는 예측 가능한 비교 결과를 반환한다.
```js
NaN === NaN;			// false
isNaN(NaN);				// true
Object.is(NaN, NaN);	// true

0 === -0;				// true
Object.is(-0, +0);		// false
```
