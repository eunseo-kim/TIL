# 📝타입 변환(9장-①) / TIL(21.03.16)

### 📌 타입 변환이란?

- 명시적 타입 변환(타입 캐스팅) : 의도적으로 값의 타입을 변환하는 것
- 암묵적 타입 변환(타입 강제 변환) : 개발자의 의도와는 상관 없이 자바스크립트 엔진에 의해 암묵적으로 타입이 변환되는 것

---

### 📌 암묵적 타입 변환

1. 문자열 타입으로 변환

   - `+` : 피연산자 중 하나 이상이 문자열이면 문자열 연결 연산자로 동작한다.
   - ES6에 도입된 템플릿 리터럴의 표현식 삽입은 표현식의 평가 결과를 문자열 타입으로 암묵적 타입 변환한다.

   ```js
   1 + '2'; // "12"로 변환됨
   `1 + 1 = ${1 + 1}` // "1 + 1 = 2"로 변환됨
   ```

2. 숫자 타입으로 변환 

   - `*`, `-`, `/`와 같은 산술 연산자의 경우, 피연산자 중에서 숫자 타입이 아닌 피연산자를 숫자 타입으로 암묵적 타입 변환한다.
   - **피연산자를 숫자 타입으로 변환할 수 없는 경우, ** 평가 결과는 `NaN`이다.
   - `+` 단항 연산자도 숫자 타입으로 암묵적 타입 변환을 수행한다.

3. 불리언 타입으로 변환

   - **자바 스크립트 엔진은 불리언 타입이 아닌 값을 Truthy(true로 암묵적 타입 변환됨)값 또는 Falsy(`false`, `undefined`, `null`, `0`, `-0`, `Nan`, `''`)값으로 구분한다.**

---

### 📌 명시적 타입 변환

1. 문자열 타입으로 변환
   - String 생성자 함수를 new 연산자 없이 호출 → ex. `String(1);` 
   - Object.prototype.toString 메서드를 사용하기 → ex. `(1).toString();`
   - 문자열 연결 연산자를 이용하기 → ex. `1 + '';`

2. 숫자 타입으로 변환

   - Number 생성자 함수를 new 연산자 없이 호출
   - parseInt, parseFloat 함수를 사용(문자열을 숫자 타입으로)
   - `+` 단항 산술 연산자 이용하기 → ex. `+'0';`

   - `*` 산술 연산자를 이용하기 → ex. `'0' * 1;`

3. 불리언 타입으로 변환

   - Boolean 생성자 함수를 new 연산자 없이 호출

   - `!` 부정 논리 연산자를 2번 사용하기 → ex. `!!0`, `!!Nan`, `!!{}`

     > 참고로... `!!{};`, `!![];` 같이 객체 타입을 불리언 타입으로 바꾼 것의 결과는 `true`이다.
