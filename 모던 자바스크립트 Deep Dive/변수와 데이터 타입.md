# 📝자바스크립트의 변수와 데이터 타입 / TIL(20.02.27)
## 📌 변수
> 변수 : 값의 위치를 가리키는 상징적인 이름
- 변수를 선언한 이후 변수에 값을 할당하지 않으면, 자바스크립트 엔진은 확보된 메모리 공간에 `undefined`라는 값을 암묵적으로 할당한다.
- 변수에 값을 재할당하면 새로운 메모리 공간을 확보하여 그 메모리 공간에 값을 저장한다. 그 이전의 값은 아무도 사용하고 있지 않으면 가비지 콜렉터에 의해 자동 해제된다.

## 📌 var의 단점
- 여러 단점이 있지만 그 중 가장 대표적인 것이 '블록 레벨 스코프를 지원하지 않고 함수 레벨 스코프를 지원'한다는 것이다.
- 따라서 의도치 않게 전역 변수가 선언되어 심각한 부작용이 발생하기도 한다.
- ES6 이후에  `let`, `const` 키워드가 도입되었다. 여전히 `var`은 사용가능하지만 권장하지는 않는다.

## 📌 변수 선언 실행 시점과 변수 호이스팅
> 변수 호이스팅 : 변수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징
- 변수 선언은 런타임(소스코드가 한 줄씩 순차적으로 실행되는 시점)이 아니라 그 이전 단계에서 먼저 실행된다.
```js
console.log(score); //에러가 발생하지 않고 undefined가 출력된다.
var score;
```
- 변수 선언은 런타임 이전에 먼저 실행되지만, 값의 할당은 소스코드가 순차적으로 실행되는 시점인 런타임에 실행된다.
```js
console.log(score);   //2) undefined
var score;            //1) 변수 선언
score = 100;          //3) 값의 할당
console.log(score);   //4) 100
``` 
## 📌 데이터 타입
- 자바스크립트의 숫자 타입은 배정밀도 64비트 부동소수점 형식만을 따른다. 즉, 모든 수를 실수로 처리한다.
- ES6부터 템플릿 리터럴(런타임에 일반 문자열로 변환되어 처리됨)이라고 하는 새로운 문자열 표기법이 도입되었다. 
- 심벌(ES6에 추가된 7번째 원시타입) : 변경 불가능한 원시타입으로, 다른 값과 중복되지 않는 값이므로 객체의 유효한 프로퍼티 키를 만들때 사용된다.

> ### 💡 템플릿 리터럴(\``)
> - 멀티라인 문자열 지원 : 이스케이프 시퀸스`\`를 사용하지 않고도 줄바꿈이 허용되며 모든 공백도 있는 그대로 적용된다.
> - 표현식 삽입 : `${}`으로 표현식을 감싼 후 삽입할 수 있다. `+`으로 문자열을 연결하는 것보다 가독성이 좋고 간편하다. 
---
> ### 💡 undefined와 null 타입
> - undefined : 자바스크립트 엔진이 변수를 초기화할때 사용된다. 개발자가 의도적으로 undefined를 변수에 할당하는 것은 권장되지 않는다.
> - null : 변수에 값이 없다는 것을 개발자가 의도적으로 명시할 때 사용한다. 변수가 이전에 참조하던 값을 더 이상 참조하지 않겠다는 의미이다. 

## 📌 동적 타이핑
> **동적 타이핑** : 변수가 선언이 아닌 할당에 의해 타입이 결정되고, 재할당에 의해 변수의 타입이 언제든지 동적으로 변할 수 있는 자바스크립트의 특징
- 정적 타입 언어(C, Java)는 변수를 선언할 때 데이터 타입을 사전에 선언(명시적 타입 선언)한다. 컴파일 시점에 타입 체크를 한다.
- 동적 타입 언어(js, python)는 변수를 선언할 때 데이터 타입을 선언하지 않는다. 데이터 타입의 어떠한 값이라도 자유롭게 (재)할당 가능하다.
- 동적 타입 언어는 유연성은 높지만 신뢰성은 떨어진다.

## 📌 동적 타입 언어: 변수 사용 시 주의할 점⭐
1. 변수는 꼭 필요한 경우에 한해 제한적으로 사용할 것
2. 변수의 유효 범위를 최대한 좁게
3. 전역 변수는 최대한 사용하지 않기
4. 변수보다는 상수를 이용
5. 변수 이름은 변수의 목적/의미를 파악할 수 있도록 네이밍하기
